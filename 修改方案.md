## 功能一：番剧卡片长按/右键收藏菜单

### 目标
在热门、时间线、搜索、收藏页面上，长按或右键点击番剧卡片时弹出「添加到分类」菜单，允许用户快速将番剧加入已创建的自定义收藏分类。

### 改动文件

| 文件 | 改动 |
|------|------|
| `lib/bean/widget/collect_button.dart` | 新增可选 `menuController` / `iconSize` / `constraints` / `padding` 参数；switch→map 重构 |
| `lib/bean/widget/collectable_card_wrapper.dart` (新) | 共享 `StatefulWidget`，包裹任意卡片，提供 `MenuController` + `GestureDetector` + `CollectButton` 叠加层 |
| `lib/bean/card/bangumi_card.dart` | 新增 `onLongPress` / `onSecondaryTap` 回调 |
| `lib/bean/card/bangumi_timeline_card.dart` | 新增 `onLongPress` / `onSecondaryTap` 回调 |
| `lib/pages/collect/collect_page.dart` | 始终渲染 `CollectButton`；编辑模式 = 40px 圆形，普通模式 = 32px 半透明圆形 |
| `lib/pages/popular/popular_page.dart` | 用 `CollectableCardWrapper` 包裹 `BangumiCardV` |
| `lib/pages/timeline/timeline_page.dart` | 用 `CollectableCardWrapper` 包裹 `BangumiTimelineCard` |
| `lib/pages/search/search_page.dart` | 用 `CollectableCardWrapper` 包裹 `BangumiCardV` |

### 设计要点
- `CollectableCardWrapper` 始终渲染 `GestureDetector` + `CollectButton` 在 `Stack` 中
- 按钮位置：`right: 4, bottom: 8`（32×32 半透明圆形）
- 无 `BackdropFilter`（Windows 渲染兼容性）
- 长按/右键触发 `MenuController.open()`，复用原有的分类菜单逻辑

## 功能二：收藏页更新角标

### 目标
在收藏页（追踪中）的番剧卡片上显示红色角标，提示该番剧有未观看的新剧集。

### 改动文件

| 文件 | 改动 |
|------|------|
| `lib/utils/update_check_service.dart` (新) | `UpdateCheckService` — 遍历「在看」列表，调用 Bangumi API 获取最新集数，与已记录集数比较 |
| `lib/pages/collect/collect_controller.dart` | `ObservableSet<int> bangumiIdsWithUpdate`；`@action checkForUpdates()` 含 20s 超时；`loadCollectibles()` 中延迟调用 |
| `lib/pages/collect/collect_page.dart` | 卡片右上角红点角标 |
| `lib/modules/collect/collect_module.dart` | `CollectedBangumi` 新增 `eps` 字段 (HiveField 3) |
| `lib/repositories/collect_crud_repository.dart` | 新增 `updateCollectibleEps(id, eps)` |

### 设计要点
- 使用 `BangumiApi.getBangumiEpisodesByID()`（type=0 的剧集数）而非每插件查询
- 懒加载 `_crudRepo` 避免 DI 死锁
- 15s（Service 层）+ 20s（Controller 层）超时，防止启动卡死

## 功能三：时间线卡片显示星期 + 集数

### 目标
在时间线页面的每张番剧卡片上显示该番剧的播出星期（周X）和最新集数（更新至第X话），并支持按播出日期排序。

### 改动文件

| 文件 | 改动 |
|------|------|
| `lib/bean/card/bangumi_timeline_card.dart` | 新增 `episodeCount` 参数；底部显示 📅 周X + ▶ 更新至第X话 |
| `lib/pages/timeline/timeline_controller.dart` | 新增 `ObservableMap<int, int> episodeCounts` + `fetchEpisodeCounts()` 异步拉取；`changeSortType()` 新增第 4 种排序（按 airDate） |
| `lib/pages/timeline/timeline_controller.g.dart` | MobX 代码生成 |
| `lib/pages/timeline/timeline_page.dart` | 将 `episodeCounts[item.id]` 传入 `BangumiTimelineCard` |

### 调用时机
- `getSchedules()`（当前季度）/ `getSchedulesBySeason()`（历史季度）完成后自动触发 `fetchEpisodeCounts()`
- 遍历日历中所有番剧 ID，逐个调用 `BangumiApi.getBangumiEpisodesByID()`，统计 type=0 的集数

### 星期映射
Bangumi API 返回的 `airWeekday`（1-7）映射为中文：周一 ~ 周日

### 排序选项
| 类型 | 说明 |
|------|------|
| 1 (default) | 按 ID 原始顺序 |
| 2 (score) | 按评分降序 |
| 3 (heat) | 按热度（投票数）降序 |
| 4 (air date) | 按播出日期升序 |

## 功能四：Bangumi 镜像回退

### 目标
当官方 Bangumi API 无法访问时，通过镜像 API（`api.kazumi.fyi`）获取热门番剧和时间线数据，避免页面空白。

### 改动文件

| 文件 | 改动 |
|------|------|
| `lib/request/core/dio_factory.dart` | `_BangumiMirrorInterceptor` 拦截器，重写 `api.bgm.tv` / `next.bgm.tv` → `api.kazumi.fyi` |
| `lib/request/apis/bangumi_api.dart` | 新增 `getBangumiMirrorPopularSubjects()`、`getBangumiMirrorSeasonCalendar()` 镜像端点 |
| `lib/request/config/api_endpoints.dart` | `version` → `'2.1.3'`，`apiLevel` → 7，镜像域名与路径 |
| `lib/utils/bangumi_mirror_credentials.dart` (新) | 镜像凭证 |
| `lib/utils/crypto.dart` (新) | `generateBangumiMirrorSearchSignature()` 签名算法 |
| `lib/pages/popular/popular_controller.dart` | 根据 `enableBangumiProxy` 路由到镜像 API |
| `lib/pages/timeline/timeline_controller.dart` | 根据 `enableBangumiProxy` 路由到镜像 API |
| `lib/pages/webdav_editor/webdav_setting.dart` | Bangumi 镜像开关 |
| `lib/services/storage/storage.dart` | `enableBangumiProxy` 配置键 |

### 设计要点
- 使用 **Dio 拦截器** 自动 URL 重写，与官方架构一致
- 控制器层面仍有显式路由（`_bangumiMirrorEnabled`）作为双重保障

## 杂项修复

| 问题 | 修复 |
|------|------|
| 镜像功能缺失（根本原因：缺少 `_BangumiMirrorInterceptor`） | 添加到 `DioFactory.apiDio` 拦截器链 |
| Windows 下 `BackdropFilter` 渲染异常 | 替换为 `Container` + `surfaceContainerHigh.withAlpha(180)` |
| 收藏页启动卡死 | `_crudRepo` 懒加载 + Service/Controller 双层超时 |
| 构建后收集按钮超出卡片范围 | `Positioned(bottom: 8)` 替代 `bottom: 4` |

## 构建与部署

- **Windows 构建**: `flutter build windows` → `build\windows\x64\runner\Release\kazumi.exe`
- **Android 构建**: `flutter build apk --debug` → `build\app\outputs\flutter-apk\app-debug.apk`
  - 需 Android SDK 36 + BuildTools 28.0.3
  - Gradle 代理配置在 `android/gradle.properties`（`systemProp.http[s].proxyHost/Port`）
- **分析检查**: `flutter analyze` — 零 error、零 warning
